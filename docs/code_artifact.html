PHASE 1 : 环境前置准备 节点 S1

1. 阿里云安全组配置与规划

在进行任何命令行操作前，必须通过阿里云控制台打通网络安全组策略。这是外部通过 Grafana、Prometheus 以及 ChaosMesh 进行链路状态监控与故障观测的前置大门。

Terminal Executable 
# 必须开放的阿里云入方向端口映射规则：
- TCP 80: 商城前端 (Google Online Boutique Frontend)
- TCP 3000: Grafana 可视化分析大盘
- TCP 9090: Prometheus 指标时序采集器
- TCP 2345: ChaosMesh 故障注入控制台
- TCP 30000-32767: Kubernetes 随机映射 NodePort 物理区间

成功验证标志

安全组保存成功后，阿里云控制台出入站规则列表更新，外部浏览器 telnet 47.94.7.122 对应的监控端口可连通。

历史踩坑避错

切勿忽略 30000-32767 端口的放行，K8s 集群服务通过 NodePort 暴露时必须在此物理网段内，否则会被阿里云物理防火墙无情丢包。
